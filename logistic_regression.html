Section 01

Why Binary Outcomes?

// The foundation of logistic regression

In medicine, many of the most consequential decisions are yes/no questions. Not "how much?" but "does this patient have it?" Logistic regression is the statistical engine behind these decisions.

Surgical Risk
Did the patient develop an infection at the surgical site within 30 days?
Infected / Not Infected
Readmission
Was the patient readmitted to hospital within 30 days of discharge?
Readmitted / Not Readmitted
Oncology
Based on the biopsy, is the tissue malignant or benign?
Malignant / Benign

// Why Not Linear Regression?

Linear regression can predict continuous values — like blood pressure or age. But if you ask it "what's the probability of readmission?", nothing stops it from returning 1.4 or −0.3. Probabilities must live between 0 and 1. Logistic regression enforces this with a clever transformation.

Output ∈ (0, 1) — always a valid probability

Section 03

The Risk Score (z)

// Quantifying risk with a linear equation

We don't guess where the decision line goes. We calculate a Risk Score z for every patient by combining their features with learned weights. The sign and magnitude of z tells us which side of the boundary they're on.

z = β₀ + β₁ × Age + β₂ × BloodPressure

// The Three Zones

z = 0 → Patient is exactly on the boundary (50/50 chance)
z > 0 → Patient is in the High Risk zone
z < 0 → Patient is in the Low Risk zone

Each β weight tells us how much one unit of a feature shifts the score. A larger |β₁| means age is a stronger driver of risk than blood pressure.

Section 04

The Sigmoid Squash

// Turning any score into a probability

A score of z = 4.8 means nothing in a hospital hallway. A probability of 99% chance of complication is immediately actionable. The sigmoid function performs this transformation.

σ(z) = 1 / (1 + e−z)
// Interactive Sigmoid · Drag the z-value slider
z value 0.0
50% Probability
Borderline Clinical label

// Key Properties

• When z = 0 → probability = exactly 0.5 (50/50)
• As z → +∞ → probability → 1 (certain High Risk)
• As z → −∞ → probability → 0 (certain Low Risk)
• The curve is always S-shaped — smooth, monotone, bounded

The flat "tails" of the sigmoid matter: a patient with z = 6 and z = 8 are both "essentially certain" — the model becomes less sensitive to further changes at the extremes.

Section 05

Odds Ratios & Interpretation

// Reading the results table like a clinician

The exponentiated coefficients (eβ) are Odds Ratios. They are the primary output of logistic regression in clinical papers. Learning to read them fluently is as important as knowing how to build the model.

// Results Table · Click any row for plain-English interpretation

// The Golden Rule for CIs

If the 95% Confidence Interval crosses 1.0, the odds ratio is not statistically significant at α = 0.05. The OR might look large (e.g. 1.42) but if the CI includes 1.0 (e.g. 0.88–2.31), we cannot rule out that the true effect is zero.

OR > 1 → risk factor  |  OR < 1 → protective  |  CI includes 1 → not significant

Section 06

How the Model Learns

// MLE and Log Loss explained

The model starts with random weights (β values) and iteratively improves them. The scoring function it optimises is called Log Loss — a punishment system that penalises confident wrong predictions exponentially.

// Maximum Likelihood Estimation

MLE asks: "What set of weights makes our observed training data most likely to have happened?" If a patient actually had a complication and the model predicted 0.95 probability — great. If the model predicted 0.05 for the same patient — heavy penalty.

Loss = −[y·log(p) + (1−y)·log(1−p)]

Here y ∈ {0,1} is the true label and p is the model's predicted probability. Notice: if y=1 and p→0, the loss → ∞.

Clinical datasets are simultaneously wide (many features: ICD codes, labs, meds) and small (few patients). This combination is lethal for unregularised logistic regression — the model memorises training labels and collapses on test data.

// L1 vs L2 vs Elastic Net

Ridge (L2): Shrinks all coefficients toward zero, keeps all features

Section 08

Glass Box vs. Black Box

// Why interpretability matters in clinical deployment

A deep learning model might achieve 94% accuracy on the same dataset. But when a doctor asks "why did the model flag this patient?", only one model can answer.

Logistic Regression Deep Neural Network
Interpretability✓ Odds ratios, CIs✗ Black box
Works with small n✓ Fine with <500 patients✗ Needs thousands
Probability output✓ Well-calibrated⚠ Requires calibration
Handles non-linearity✗ Linear boundary only✓ Any shape
Ethics committee answer✓ "Smoking doubles odds"✗ "The network said so"
Regulatory approval✓ Established pathway⚠ Under development
Handles 500+ features⚠ Needs regularisation✓ Naturally scales

// The Clinical Deployment Question

Logistic regression is not always the "best" model by accuracy. But it is often the most deployable model — because it can be audited, challenged in court, explained to a patient, and approved by a regulator. This is why it powers APACHE, SOFA, Wells score, and most clinical risk tools still running today.

// When NOT To Use Logistic Regression

• When the true decision boundary is highly non-linear
• When you have many interaction effects between features
• When you have tens of thousands of patients and need maximum predictive performance
• For image or sequence data (use CNNs, RNNs instead)
